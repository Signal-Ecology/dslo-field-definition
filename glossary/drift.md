# Drift (DSLO Glossary)

## Definition
Drift is the **silent, unintended shift in system meaning** that occurs even when code compiles, tests pass, and systems appear operational.

It is the primary failure mode DSLO is designed to eliminate.

## Types of Drift
- Semantic Drift  
- Invariant Drift  
- Boundary Drift  
- Spec–Code Drift  
- Context Drift  

## Causes
- AI‑accelerated refactoring  
- Unbounded context accumulation  
- Non‑deterministic agent behavior  
- Weak or implicit semantic boundaries  
- Probabilistic interpretation of domain meaning  

## Drift Consequences
- Loss of business intent  
- Misalignment between specification and code  
- Structural degradation  
- Meaning collapse over time
